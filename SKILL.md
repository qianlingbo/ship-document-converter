## 📊 单证录入规则

### Sheet 1：船员名单（非旅客人员清单）

| 字段 | 规则 |
|------|------|
| **序号** | 按顺序递增，从1开始 |
| **姓名** | 中国船员 = 中文；外国船员 = 大写英文字母 |
| **性别** | `1-男` / `2-女` |
| **船员职务** | 参数B列 → 找得到直接映射，找不到按规则分配（见下方） |
| **船员国籍** | 参数A列，格式 `CN-中国` / `VN-越南` / `MM-缅甸` 等 |
| **出生日期** | `YYYYMMDD` 格式（支持 `2024-01-15`、`2024/01/15`、`20240115` 等） |
| **出生地点** | 取国籍的中文名（如 `越南`、`缅甸`、`中国`） |
| **证件类型** | 中国船员 → `17-海员证`；外国船员 → `14-普通护照` |
| **证件号码** | 中国船员 → 海员证号码；外国船员 → 护照号码 |
| **是否申请登陆** | 留空 |
| **适任证书编号** | 留空 |
| **适任证书有效期至** | 留空 |
| **证件检查地点** | 留空 |
| **登船日期** | `YYYYMMDD` 格式，来源于 port of call 第一条记录的进港日期 |
| **登船口岸** | 参数E列，找不到保持空白 |

#### 职务分配规则（找不到时的 fallback）

```
保证有 3 个 56-高级值班水手
保证有 3 个 66-高级值班机工
其余：水手角色 → 55-值班水手
      机工角色 → 65-值班机工
```

英文缩写自动映射：
```
Master/Capt/Captain  → 51-船长
C/O                  → 52-大副
2/O                  → 53-二副
3/O                  → 54-三副
C/E                  → 61-轮机长
2/E                  → 63-二管轮
3/E                  → 64-三管轮
4/E/Fitter           → 65-值班机工
BSN/Bosun            → 55-值班水手
AB1/Able Seaman      → 56-高级值班水手
OIL1/Electrician     → 66-高级值班机工
```

### Sheet 2：船上非旅客人员物品清单

| 字段 | 规则 |
|------|------|
| **序号** | 同船员序号 |
| **证件类型** | 参照船员名单（17-海员证 / 14-普通护照） |
| **证件号码** | 参照船员名单 |
| **物品类型** | `0100`（计算机） |
| **物品名称** | `计算机` |
| **物品数量** | `1` |
| **数量单位** | `001` |
| **备注** | 留空 |

### Sheet 3：海事船岸活动信息（Port of Call）

| 字段 | 规则 |
|------|------|
| **序号** | 按顺序递增 |
| **进港时间** | `YYYY/MM/DD HH:MM:SS`，时间随机取 00:00–12:00 |
| **离港时间** | `YYYY/MM/DD HH:MM:SS`，时间随机取 12:00–24:00（同一天） |
| **国家/地区名称** | 参数A列，自动从港口名提取（如 `JP-日本`） |
| **船舶保安等级** | `1-1级` |
| **特别/附加保安设施** | 留空 |
| **停靠港口** | 参数E列，找不到保持空白 |
| **港口保安等级** | `1-1级` |

## 🔧 核心函数说明

### `normalize_code(val, NATIONALITY_MAP)` → `CN-中国`
模糊匹配国籍：支持代码（`VN`）、全称（`VIETNAM`）、中文（`越南`），自动规范化到参数格式。

### `match_duty(val)` → `51-船长`
模糊匹配职务：支持中文职务名、英文缩写、标准代码。

### `match_port(val)` → `CNXAM-厦门`
模糊匹配港口：支持中英文港口名、国家+港口组合（`LIANYUNGANG, CHINA`）。

### `normalize_date(val)` → `20240115`
支持 `datetime` 对象、`YYYY-MM-DD`、`YYYY/MM/DD`、`YYYYMMDD`、`DD/MM/YYYY` 等格式。

### `get_country_name_for_port(port_val)` → `JP-日本`
从港口名自动提取国家：优先从原始值（如 `HITACHINAKA, JAPAN`）中提取国家部分，再查港口码对应国家。

## 📁 输入文件格式

### IMO Crew List（Excel）

典型表头含关键词：`No.` + `Family name` + `Rank`（第1行或第10行均可自动识别）

列布局：
```
B=序号  C=姓名  D=性别  E=职务  F=国籍  G=出生日期  H=出生地点
I=护照号码  J=护照有效期  K=登船日期  L=登船地点
```

### Port of Call List（Excel）

典型表头含关键词：`Voy.` + `Port`（第1行或第7行均可自动识别）

列布局：
```
B=Voy.No  C=Port  D=Arrival  E=Departure  F=Cargo
G=Port Security Level  H=Ship Security Level  I=Additional Security
```

## 📌 已知局限

1. **PDF 支持**：基础实现，需根据实际 PDF 布局调整列索引
2. **护照有效期 / 适任证书**：留空，未从原始文件提取
3. **物品清单**：固定为每人一台计算机，可按需调整
4. **多国籍混合船员**：按同船大多数船员判断（见源码 `_raw_nation` 逻辑）
